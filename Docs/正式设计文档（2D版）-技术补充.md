# 真名剥露 · 正式设计文档（2D版）

**代号**：TrueName Unveiled
**引擎**：团结引擎 1.9.3（Apple Silicon）
**渲染**：Built-in 2D（30h 简化，效果与 URP 2D 等效）
**分辨率**：1920 × 1080 固定横版
**目标时长**：单局 3–5 分钟

## 四、核心机制

### 4.1 双语言弹咒与风险应对
- 单词从右侧飞入，水平向左，带轻微上下正弦波动（振幅 10px，频率 2Hz）。
- 每个单词携带 `LanguageType`：Sacred（圣文/意文/英文混用）或 Demonic（魔文/德文），并分为两类：
  - **普通词**（金/紫）：必须处理，放跑会 +暴露度。
  - **干扰词**（灰色裂纹，边缘发光）：应放跑；命中会 +8% 暴露，不揭示字母。
- 输入判定：
  - `Q` 圣盾：
    - 圣文单词：正确，免费揭示 1 个字母，不消耗圣光。
    - 魔文单词：消耗圣光，抵消该词（不揭示、不惩罚）。
    - 干扰词：消耗圣光，抵消该词（不惩罚、不揭示）。
  - `E` 魔文刃：
    - 魔文单词：正确，免费揭示 1 个字母，并叠加 1 层“剥露”。
    - 圣文单词：错误，Miss + 暴露度。
    - 干扰词：错误，Miss + 8% 暴露度。
  - E 判定窗口为单词停留全时长（0.5s），而非 0.2s 窄窗。
  - 放跑普通词：视为 Miss，+暴露度。
  - 放跑干扰词：无惩罚，鼓励识别并放弃。
- 单词进入防御线 `X=640` ±50px 范围时触发判定，吸附到 `X=640` 后停留 0.5s。
- 击碎后单词爆为对应颜色粒子（2D Particle，Burst 发射，每次 15 个）。

### 4.2 真名拼图与剥露层数
- 敌人拥有一条 6 字母真名（如 `MIRROR`），显示在屏幕顶部空槽中。
- 每个空槽初始为 `_`，揭示后显示对应字母。
- 普通词被成功处理时揭示字母：
  - 圣文 + Q：揭示 1 字母，+8% 领域能量。
  - 魔文 + E：揭示 1 字母，+12% 领域能量，并叠加 1 层“剥露”。
- 剥露层数：
  - 上限 3 层，UI 在敌人剪影上方显示 3 个紫黑小点。
  - 叠满 3 层时，强制额外揭示 1 个未揭示字母，并恢复 5% 暴露度；层数清零。
  - 额外揭示也加 12% 领域能量。
- Q 圣盾抵消不揭示、不叠加剥露。
- 玩家随时可按 `Enter` 打开猜名输入框；输入正确真名直接触发胜利演出，输入错误 +25% 暴露度。
- 当所有字母已揭示时，仍需要手动输入真名确认，防止误触胜利。

### 4.3 连携与双诵
- 计数器：连续成功处理单词（圣文+Q、魔文+E、或 Q 抵消魔文/干扰词），UI 右上角显示连击数。
- 双诵触发：连续成功 3 次后，下一个普通词强制为双色词（金紫渐变）。
- 输入：双色词需 `Q` 和 `E` 在 **0.15 秒** 内都按下。
- 成功：揭示一个字母，+15% 领域充能，+1 剥露层数，屏幕震动 0.3 秒。
- 失败：按错或超时，视为普通 Miss，暴露度 +15%。

### 4.4 圣光资源
- 圣光值 0–100%，初始 100%，自然回复约 5%/秒。
- 每次按 `Q` 抵消魔文/干扰词时消耗 20% 圣光。
- 圣文单词按 Q 不消耗圣光（正确语言免费揭示）。
- 圣光降至 0 时进入 3 秒枯竭，期间无法使用 Q 抵消。
- UI 显示在底部资源条左侧。

### 4.5 领域展开（SPACE）
- 充能：每次成功揭示字母 +8%（Q 圣文）/ +12%（E 魔文/额外揭示）；双诵成功 +15%。
- 释放：满能量（100%）后按 `Space`。
- 演出：
  - `Time.timeScale` 从 1 → 0.1，耗时 0.3 秒。
  - 巫王背后三层圣环扩张。
  - 场上所有现存单词自动击碎。
  - 所有未揭示字母全部显示 3 秒（猜名窗口）。
  - 玩家暴露度 -10%。
  - 3 秒后 `Time.timeScale` 恢复，字母重新隐藏。
- 可多次释放（无 `used` 限制）。

## 五、攻击模式与阶段（黑化镜像）

### 5.1 攻击模式表

| 模式 | 名称 | 生成频率 | 单词数量 | 飞行速度 | 成功效果 | 失败惩罚 |
|------|------|----------|----------|----------|----------|----------|
| A | 单发突刺 | 每 4 秒 | 1 个 | 120px/s | 揭示 1 字母 | 暴露 +4% |
| B | 三连斩 | 每 15 秒 | 3 个连续，间隔 0.4 秒 | 180px/s | 每下揭示 1 字母 | 每个单词独立判定，单次 Miss +8% |
| C | 祷言蓄力 | 每 30 秒 | 1 个长单词（6-8 字符） | 80px/s（极慢） | 揭示 2 字母 | 暴露 +15% |

干扰词规则：
- 所有模式中，每个单词有概率为干扰词（灰色裂纹）：阶段一 20%、阶段二 40%、阶段三 60%。
- 干扰词被 Q/E 命中时 +8% 暴露，不揭示字母；放跑无惩罚。

### 5.2 三阶段转换

| 阶段 | 揭示字母数 | 时长 | 可用模式 | 干扰词概率 | 背景变化 | 镜像演出 |
|------|------------|------|----------|------------|----------|----------|
| 一 | 0–2 | 60 秒 | 仅 A | 20% | 静态紫黑 | 正常呼吸 |
| 二 | 3–4 | 90 秒 | A + B | 40% | 背景变暗，边缘起雾 | 剪影出现裂纹发光，呼吸加快 |
| 三 | 5–6 | 60 秒 | A + B + C | 60% | 背景出现裂纹，圣光从顶部漏下 | 全身剧烈抖动，眼瞳变红 |

阶段转换演出：
- 揭示字母数跨阶段阈值时，时间冻结 0.5 秒。
- 镜像剪影发出阶段台词（见第七节）。
- 背景色在 1 秒内渐变。

## 十一、技术实现要点

### 11.1 相机与分辨率
- 固定横版 1920×1080，主相机 `Orthographic`，Size 5.4。
- Canvas 使用 `Screen Space – Overlay`，Reference Resolution 1920×1080，Match 0.5。
- 无摄像机跟随，无场景滚动。

### 11.2 输入系统
- 所有输入走 `InputManager`。
- 维护当前处于防御线的单词队列（通常只有 1 个）。
- `Q` 短按：触发圣盾判定（圣文免费揭示 / 魔文/干扰词消耗圣光抵消）。
- `E` 短按：触发魔文刃判定（仅魔文正确，判定窗口为单词停留全时长）。
- `Enter`：打开猜名输入框（Combat 状态），时间减速到 0.2，弹幕继续移动。
- `Space`：触发领域展开（需满能量）。
- 双诵词用时间戳缓冲：记录最近 `Q` 和 `E` 按下时间，进入防御线时检查是否都在 0.15s 内按下。

### 11.3 单词飞行状态机
```csharp
public enum WordState { Fly, Hold, Miss, Hit }
public enum WordPattern { A, B, C }
public enum WordType { Normal, Interference }
```
- `Fly`：`transform.Translate(Vector3.left * speed * Time.deltaTime)`，叠加正弦波动。
- 到达 `X <= 640 + 50` 进入 `Hold`，并吸附到 `X = 640`。
- `Hold` 期间被处理 → `Hit`：
  - 圣文词 + Q：揭示字母，不耗圣光。
  - 魔文词 + E：揭示字母，+1 剥露层数。
  - 魔文词/干扰词 + Q：消耗圣光，抵消（不揭示、不惩罚）。
  - 圣文词/干扰词 + E：Miss，+8% 暴露（干扰词）或普通暴露（圣文词）。
  - 双诵词需 Q 和 E 在 0.15s 内同按。
- `Hold` 结束仍存活 → `Miss`：
  - 普通词 → +暴露度。
  - 干扰词 → 无惩罚。
- 使用对象池，不要直接 `Destroy`。

### 11.4 真名拼图与剥露层数
- `TrueNameSystem` 维护敌人真名字符串（6 字母）和 `revealedMask` 位掩码。
- `RevealLetter()`：随机选择未揭示位，更新掩码，通知 UI，并增加领域充能。
- `AddExposureLayer()`：E 魔文刃成功时调用，叠加剥露层数。满 3 层时额外揭示 1 个未揭示字母，恢复少量暴露度，清零层数。
- `GuessTrueName(string guess)`：
  - 正确：触发胜利。
  - 错误：+25% 暴露度，台词嘲讽。
- 玩家暴露度、领域能量用 `float` 存储，更新时 `Mathf.Clamp`。
- 使用 `UnityEvent` 通知 UI 和阶段系统：
  - `OnLetterRevealed`
  - `OnExposureLayerChanged`
  - `OnExposureChanged`
  - `OnDomainChargeChanged`
  - `OnWin`
  - `OnLose`

### 11.5 阶段转换
- `PhaseManager` 监听 `OnLetterRevealed`，按揭示字母数 0–2 / 3–4 / 5–6 切换阶段。
- 切换时触发：背景色渐变、台词、镜像裂纹/眼瞳变化、干扰词概率变化。
- 时间冻结 0.5s 可用 `Time.timeScale = 0` + `WaitForSecondsRealtime`。

### 11.6 领域展开
```csharp
IEnumerator DomainExpand() {
    Time.timeScale = 0.1f;
    wordSpawner.ClearAllWords();
    trueNameSystem.RevealAllLettersTemporary(3f);
    trueNameSystem.ExposePlayer(-10f);
    OnDomainExpanded?.Invoke();
    DomainVisual.Instance?.Play();
    yield return new WaitForSecondsRealtime(3f);
    Time.timeScale = 1f;
    trueNameSystem.HideAllLetters();
}
```
- 慢动作期间，UI 闪光、立绘呼吸、屏幕震动需用 `Time.unscaledDeltaTime`。
- 领域展开期间所有字母临时显示，3 秒后恢复隐藏，给玩家猜名窗口。
- 粒子系统可勾选 `MainModule.useUnscaledTime`。
- 无使用次数限制，但需要重新充能。

### 11.7 屏幕震动与侵蚀
- 震动：给相机 Transform 加短协程，用 `Mathf.PerlinNoise` 或随机偏移，结束后归零。
- 侵蚀：全屏 `Image` 紫黑渐变，按暴露度插值 Alpha。
- 碎镜：Game Over 时切换裂纹贴图，从中心放大，粒子 Burst 一次。

### 11.8 TextMeshPro 与字体
- 动态单词用 `TextMeshProUGUI`。
- 中文思源宋体生成 SDF Atlas（700+ 常用字）或使用动态字体。
- 意文/德文使用 Cinzel、UnifrakturMaguntia。
- 建议除颜色外，给单词加不同 Outline 或 Icon 前缀，色盲友好。

### 11.9 性能与散热（MacBook Air M3）
- 对象池预生成 20–30 个单词。
- 粒子总数锁死 500 以内；击碎用 `Burst`，别用持续发射。
- 避免 `Rigidbody2D`、`OnCollisionEnter2D`、复杂 Shader。
- 每阶段只切换一次材质/颜色，不要每帧 `Lerp` 全局变量。
- 每编译/Play 30 分钟歇 5 分钟。

### 11.10 构建配置
- 平台：macOS（Apple Silicon）。
- 图形 API：Metal。
- 分辨率：Player Settings → Resolution → 固定 1920×1080，保持宽高比。
- Day 3 最后 2 小时必须跑一次完整 Mac Build，检查字体、TMP、Sprite 导入和输入焦点。
